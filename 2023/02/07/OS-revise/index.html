# CH1. 操作系统引论

操作系统的

# 目标：

① 方便性 ② 有效性 ③ 可扩充性 ④ 开放性

# 作用：

    1. 作为用户与计算机硬件系统之间的接口
    1. 作为计算机系统之间的管理者
    1. 实现对计算机资源的抽象

# 特征：并发、共享、虚拟、异步

虚拟：操作系统会把一个物理上的实体变为若干个逻辑上的对应物，物理实体是实际存在的，而逻辑对应物是用户感受到的.
    “空分复用技术” (让内存看起来很多) & “时分复用技术” (让CPU看起来很多)

异步：在多道进程运行的环境下，进程之中指令的执行不是一贯到底的，而是走走停停，以不可预知的速度向前推进，这就是异步性。

# 功能：

# 处理机管理功能：

创建和撤销进程,对各进程的运行进行协调,实现进程之间的交换以及按照一定的算法把处理机分配给进程。

# 1. 进程控制

 为作业创建进程,撤销(终止)已结束的进程,以及控制进程在运行过程中的转换。

# 2. 进程同步

 主要是对多个进程(线程)的运行进行协调。两种协调方式。

 1).进程互斥方式
    各进程对临界资源访问时,采用...
    如锁,锁开可以访问 锁关禁止访问~

 2).进程同步方式
    在相互合作以完成共同任务的各进程间,由同步机构对他们的执行次序加以协调。
    如信号量机制。

# 3. 进程通信

 实现相互合作进程之间的信息交换。

# 4. 调度

 1).作业调度
 用某种算法从后备队列中选出若干个作业,并为他们分配所需的资源;在将这些作业调入内存后,分别为他们建立进程,使他们成为可能获得处理机的ready进程,并将这些进程插入就绪队列。
 2).进程调度
 按照一定的算法 从进程的就绪队列中 选出一个进程,将处理机分配给他,并为他设置运行现场,使其投入运行。

# 存储器管理功能：

# 1. 内存的分配与回收

 分配-> 静态分配 & 动态分配

 回收 程序所占用的内存,并根据当前的内存管理算法将回收的内存 经过处理放入对应的 管理数据结构中,供下次分配使用。

# 2. 内存保护

 1)确保每道程序都仅在自己的内存空间运行,彼此互不干扰。
 2)绝不允许 用户程序访问OS的 程序和内存,也不允许转移到 非共享的其他用户中去执行。

一种简单の内存保护机制：设置两个界限寄存器,分别用于存放正在执行的上届和下届。
在程序运行时,系统对每条指令所要访问的地址进行检查,如果发生越界,则发出越界中断请求,已停止该程序的执行。

# 3. 地址映射

 地址映射功能,能够将地址空间中的逻辑地址 变换成 内存空间中与之对应的 物理地址。(要在硬件功能支持下实现)

# 4. 内存扩充

 借助虚拟存储技术,在逻辑上扩大内存,实现->
 1).请求调入功能：

 系统允许 在仅装入部分 用户程序和数据的情况下 启动该程序运行。
 在程序运行过程中,若发现 继续运行所需的 程序和数据尚未装入内存,可向OS发出请求,OS会从磁盘中将 所需部分 调入内存。

 2).置换功能：
 若发现内存已无足够空间来装入所需~时,系统能将内存中暂时不用的~调至盘上,腾出空间,再。

# 设备管理功能：

1).完成用户进程提出的I/O请求,位用户进程分配所需的I/O设备 并完成指定的操作 2).提高CPU和I/O设备的利用率,提高IO速度

# 1. 缓冲管理

 有效地缓和CPU I/O 设备速度不匹配的矛盾,提高CPU的利用率,进而提高系统的吞吐量。

 常见的BUFFER机制: 1.单缓冲区机制；2.能实现双向同时传送数据的双缓冲区机制；3.能提供多个设备同时使用的 公用缓冲池机制

# 2. 设备分配

 根据用户进程的I/O请求与系统现有资源情况,按照某种~策略,为之分配所需设备。

# 3. 设备处理

 又称 设备驱动程序,
 是实现CPU和设备控制器之间的通信,
 即 由CPU向设备控制器 发出I/O命令,要求他完成指定的I/O操作；
 以及由CPU接收从设备控制器发来的中断请求,并给予迅速的相应及相应的处理。

# 文件管理功能：

# 1. 文件存储空间管理

 为每个文件分配必要的外存空间,提高外存的利用率、文件系统的存取速度。

# 2. 目录管理

 为每个文件建立一个目录项,目录项包括文件名、文件属性、文件在磁盘上的物理位置,并对众多的目录项加以有效的组织,以实现按名存取。
 还应实现 文件共享(在外存上保留那副本)、快速的目录查询手段。

# 3. 文件的读 / 写管理和保护

 - 1.文件的读/写管理

 根据用户的请求 从外存中读取数据,or将数据写入外存。
 读写操作不会同时进行,可合用一个读写指针。

 - 2.保护

 防止系统中的文件 被非法窃取&破环,目标->
 1)防止未经核准的用户存取文件
 2)防止冒名顶替存取文件
 3)防止以不正确的方式使用文件

# 特征 --- 单道批处理系统、多道批处理系统、分时系统、实时系统。

# 单道批处理系统

系统对作业的处理是成批进行的，但内存中始终只运行一道作业

自动性(磁带上的一批作业能自动运行，无需人工干预)
顺序性(磁带上的各道作业按顺序进入内存,先调入内存的作业先运行)
单道性(内存中仅运行一道作业)

# 多道批处理系统

允许多个程序同时进入内存并运行。当一道程序因 I/O 请求而暂停时，CPU 便立即转去运行另一道程序。达到了系统各部件之间的并行工作，提高资源利用率，增大了系统吞吐量。

多道:计算机内存中存放多道相互独立的程序
宏观上并行：同时进入系统的 多道程序都处于运行过程中，
		即 它们先后开始了各自的运行，但都未运行完毕。
微观上串行：内存中的多道程序轮流占有CPU，交替执行。

# 分时系统

比如信息查询系统
1）人机交互 2）共享主机

# (1) 多路性：

系统允许将多台终端同时连接到一台主机上,并按分时原则为每个用户提供服务。
允许多个用户共享一台主机,提高了资源利用率,降低了费用,促进了jsj广泛应用。

# (2) 独立性：

指系统提供了一种环境,即 每个用户在各自的终端上进行操作,彼此间互不干扰,给用户的感觉就是他一人独占主机。

# (3) 及时性：

用户的请求能在很短的时间内获得相应。

这一时间间隔是人们能接受的，通常 1~3 秒。

# (4) 交互性：

用户可以通过终端与 系统进行广泛的人机对话。

表现在：用户可以请求系统进行多方面的服务，如文件编辑和数据处理，访问系统中的文件系统和数据库系统，打印运行结果等。

# 实时系统

# (1) 多路性：

系统周期性的对多路现场信息进行采集,并对多个对象or执行机构进行控制

# (2) 独立性：

对信息的采集&对 对象的控制也是彼此互不干扰的

# (3) 及时性：

多媒体系统实时性 要求播出来的音频能令人满意。
实时控制系统的实时性则是以控制对象所要求的截止时间来确定的,一般为秒级~毫秒级。

# (4) 交互性：

在信息查询系统中,人与系统的交互性仅限于访问系统中的某些特定的专用服务程序。
它并不能像 分时系统那样向终端用户提供 数据处理、资源共享等服务.
多媒体系统的交互性也仅限于 由用户发送某些特定的指令,如开始、停止、快进、然后由系统立即响应。

# (5) 可靠性:

分时系统要求可靠，
实时系统要求高度可靠,因为任何差错都会带来无法预料的灾难性后果.
因此,在实时系统中,往往都采取了多级容错措施,以保障系统及数据安全。

# 概念的区别 --- 并发与共享

# 并发

-> 并行与并发
并行指 两个或多个事件在同一时刻发生；并发 ：~ 在同一时间间隔内发生

在多道程序运行环境下，并发是指一段时间内宏观上有多个程序在运行；
但在单处理机系统中，每一时刻仅能有一道程序执行，故在微观上这些程序只能分时交替执行。

并发：同一时间段内，系统可以运行多个进程。
并发 = 宏观上并行 + 微观上串行
我们 QQ,WX, 音乐。。。看来是 同时运行的 (宏观)
实际上，3 个进程是不断的切换运行的，CPU 按照某种算法切换进程 (围观)

# 共享

OS 中的资源 可供内存中的 多个并发运行的进程 共同使用。(资源复用)

宏观上既限定了时间(进程在内存中时),又限定了地点(内存)
同时使用QQ,WX，它们是并发运行的，可以共享的访问硬盘资源
    1. 互斥共享：

当进程 A 要访问资源时，必须先提出请求。
若此时该系统资源空闲，系统便可将其分配给进程 A 使用。
此后若再有其它进程要访问该资源，只要 A 未用完，其它进程必须等待。
仅当 A 访问完并且释放后，才允许另一进程对该资源访问。

摄像头资源是互斥共享的。在同一时间段，摄像头资源只能分配给一个进程。

    1. 同时共享：
      （‘同时’- 宏观）在同一时间段内，磁盘资源可以供多个进程使用。

并发与共享是多用户 OS 最基本两个特征。
互为对方存在的条件，
一方面，资源共享是以进程的并发执行为条件的，若系统不允许并发执行，则不存在资源共享问题。
另一方面，若系统不能对资源共享实施有效的管理，以协调好各进程对共享资源的访问，则必然会影响各进程间并发执行的程度，甚至根本无法并发执行。

# 系统调用的概念

add: head {
程序接口 是 OS 专门为 用户程序设计的，是用户程序取得 OS 服务的唯一途径。
程序接口由一组 系统调用 (system call) 组成，
系统调用 提供了 用户程序和 OS 内核之间的接口。

每一个系统调用都是一个 能完成特定功能的子程序。
}

系统调用是 OS 内核与用户程序之间的接口，每个 OS 都提供了大量的系统调用给程序员使用。

目的：使应用程序可以通过系统调用来间接调用 OS 中的 相关过程，进而取得相应的服务。

系统调用在本质上是 应用程序请求 OS 内核完成某功能时的 一种过程调用，但它 --->

是一种特殊的过程调用：-> 与一般的过程调用区别在 ->

(1) 运行在不同的系统状态
一般的过程调用：
调用程序 & 被调用程序 运行在相同的状态，内核态 or 用户态
系统调用：
调用程序运行在用户态，被调用程序运行在内核态。

(2) 状态的转换
一般的过程调用：
不涉及系统状态的转换，可直接由 调用过程转向被调用过程。
系统调用：
调用过程 & 被调用过程处于工作处于不同的系统状态，
因而不允许由调用过程直接转向被调用过程，
需要通过 软中断机制 先由用户态转为内核态，
经内核分析后才能转向相应的系统调用处理子程序。
(3) 返回问题

(4) 嵌套调用

# 进程的 含义、特征、组成

# 定义

进程实体 (进程映像)：程序段、数据段、PCB。构成
进程实体 简称 进程，

如 创建进程，实质上是指创建进程的 PCB; 撤销进程，实质上是指撤销进程的 PCB

典型 3 种定义：
1). 进程是程序的一次执行。
2). 进程是一个 程序及数据 在处理机上执行时所发生的活动。
3). 进程是具有独立功能的程序 在一个数据集上执行的过程，它是系统进行资源分配和调度的一个独立单位。

传统 OS 中的进程定义:
进程是 程序的执行过程，是 系统进行资源分配和调度的一个独立单位。

# 特征

(1) 动态性
(2) 并发性
(3) 独立性
(4) 异步性

    1. 动态性:

    进程的实质 is 程序的执行过程，so, 动态性是进程最基本的特征。
    表现在：进程由创建而产生，由调度而执行，由释放而消亡。

    可见，进程有一定的生命周期、而程序只是一组有序指令的组合，并存放于某种介质上，其本身并没有活动的含义，因而是静态的。

    1. 并发性:

    指多个进程共存于内存中，且能在一段时间内同时执行。
    引入进程是为了 使进程能和其它进程并发执行。
    thus, 并发性 is 进程、OS の重要特征。;
    而程序 (未建立 PCB) 是不能参与并发执行的。

    1. 独立性:

    进程是一个能够独立运行、独立获得资源、独立接受调度的基本单位。
    凡未建立 PCB 的程序都不能作为一个独立的单位参与并发执行

    1. 异步性:

    进程是按异步方式执行的，即按各自独立，不可预知的速度向前推进。

# 3 基本状态

就绪：
已处于准备好执行的状态，
进程已分配到除 CPU 以外的所有必要资源 (内存空间、I/O 等) 后，只要再获得 CPU, 便可立即执行。
如果有许多 -> 优先级策略 -> 队列 -- 就绪队列。

执行：

对任一时刻，
单处理机系统中，只有一个进程处于执行状态；
多处理机系统中，可能会有多个进程处于执行状态。

阻塞：
正在执行的进程由于发生某些事件 (如 I/O 请求、申请缓冲区失败..)
而暂时无法继续执行。
会引发 进程调度，
OS 会把处理机分配给另一个进程，而让受阻进程处于暂停状态 [- 阻塞状态、等待状态、封锁状态]

# 线程的基本概念

序言：
OS 引入进程 - 使多个程序能并发执行 - 提高资源利用率 & 系统吞吐量
OS 引入线程 - 减少程序在并发执行时所付出的时空 (时间 & 空间) - OS 有更好的并发性

回顾：

  1. 进程的 2 个基本属性：
    1).is a 可拥有资源的独立单位。
    2).is a 可独立调度和分派的基本单位。(dispatch assign)

  2. 程序并发执行所需付出的时空开销
    1). 创建进程 - 建立相应 PCB
    2). 撤销进程 - 先对其占有的资源执行回收，再撤销 PCB。
    3). 进程切换 - 上下文切换时，保留当前进程的 CPU 环境，设置新的 CPU 环境，花费很多处理机时间。
    进程是一个资源的拥有者，.. 付出较大的时空开销、。

线程 ：作为调度和分派的基本单位
线程定义：

线程是进程内一个相对独立的可调度的执行单元。

线程，有时被称为轻量进程 (Lightweight Process，LWP），是程序执行流的最小单元。

线程，是进程的一部分，一个没有线程的进程可以被看作是单线程的。线程有时又被称为轻权进程或轻量级进程，也是 CPU 调度的一个基本单位。

# 区别 - 进程 & 线程。

1：线程定义

线程是进程的基本执行单元，一个进程的所有任务都在线程中执行

进程要想执行任务，必须得有线程，进程至少要有一条线程

程序启动会默认开启一条线程，这条线程被称为主线程或 UI 线程

2：进程定义

进程是指在系统中正在运行的一个应用程序

每个进程之间是独立的，每个进程均运行在其专用的且受保护的内存

3：进程与线程的区别

  1. 调度的基本单位
传统OS,进程作为 独立调度和分派的基本单位,能够独立运行。每次调度-上下文切换,开销较大。
引入线程的OS,线程作为 调度和分派的基本单位,线程是能够独立运行的基本单位。线程切换-仅需保存和设置少量寄存器的内容,切换代价远低于进程。
  1. 并发性
引入线程的OS,
进程之间、一个进程的多个线程之间、一个进程的所有线程之间、
不同进程的线程之间
都能并发执行。
提高OS的并发性、资源利用率、系统吞吐量。

地址空间：同一进程的线程共享本进程的地址空间，而进程之间则是独立的地址空间。

  1. 拥有资源：
线程几乎不拥有资源,有TCB,
同一进程的所有线程都具有相同的地址空间,线程可以访问该地址空间中的每一个虚地址。
同一进程内的线程共享本进程的资源如内存、I/O、cpu等，但是进程之间的资源是独立的。
  1. 独立性
同一进程中的不同线程之间的独立性 比不同进程之间的独立性低得多。 
一个进程崩溃后，在保护模式下不会对其他进程产生影响，但是一个线程崩溃整个进程都死掉。所以多进程要比多线程健壮。

执行过程：每个独立的进程有一个程序运行的入口、顺序执行序列和程序入口。但是线程不能独立执行，必须依存在应用程序中，由应用程序提供多个线程执行控制。
  1. 系统开销

进程切换时，消耗的资源大，效率高。所以涉及到频繁的切换时，使用线程要好于进程。同样如果要求同时进行并且又要共享某些变量的并发操作，只能用线程不能用进程

  1. 支持多处理机系统
单线程进程,不管有多少处理机,该进程只能运行在一个处理机上。
多线程进程,一个进程中的多个线程可分配到多个处理及上,
并行运行,加速进程的完成。

# 共享存储器系统

1). 基于共享数据结构的通信方式

各进程共享某些数据结构,以实现各进程的信息交换,如 生产者-消费者中的 有界缓冲区。
仅~较少数据、通信效率低、*低级*。

2). 基于共享存储区的通信方式

为传送大量数据,内存中划出了一块共享存储区,
各进程可通过对该共享存储区的读/写来交换信息、实现通信,
数据的形式和位置(甚至访问)均由进程负责控制,而非OS。
高级。
需要在通信前,先向系统申请获得共享存储区中的一个分区,并将其附加到自己的地址空间中,进而便可对其中的数据进行正常的读/写,读/写完成or不再需要时,将分区归还给共享存储区即可。

# 管道通信系统

指用于连接一个读进程和一个写进程以实现它们之间通信的一个共享文件，又名 pipe 文件。
向管道 (共享文件) 提供输入的写进程，会以字节流形式将大量的数据送入管道；
而接收管道输出的读进程，则会从管道中读数据。

首创 UNIX 系统。

必须提供以下 3 方面の协调能力：

1).互斥, 当一进程正在读/写时,其他进程必须等待。

2).同步, 

当写进程把一定数量的数据写入管道后,便去睡眠,直到读进程把数据读走后,再把它唤醒。
当读进程读一空管道时,便去睡眠,直至写进程将数据写入管道时,再把它唤醒。

3).确定对方是否存在。

只有确定了对方已存在时,才能进行通信。

# 消息传递系统

不必借助任何数据结构 or 存储区，
-> 以格式化的消息 (message) 为单位，将通讯的数据封装在消息中，
并利用 OS 提供的一组通信命令 (原语), 在进程间进行消息传递，完成进程间的数据交换。

- 通信过程对用户透明，降低复杂性、错误率，应用广泛；
支持 多处理器系统、分布式系统、计算机网络系统。

1). 直接通讯方式
发送进程直接使用 OS の发送原语把消息发送给目标进程。
2). 间接通讯方式
发送进程和接收进程都通过共享中间实体 (信箱) 的方式 进行消息的发送和接收，进而完成进程间的通信。

# 客户机 - 服务器系统

主流！
实现方法 3 类：socket、远程过程调用、远程方法调用。

    1. Socket

    UNIX.
    一个通信标志类型的数据结构，
    包含 通信目标地址、端口号、传输层协议、进程所在的网络地址、针对客户 or 服务器程序所提供的不同系统调用 or API 等，
    是 进程通信 and 网络通信的基本构件。

    分 2 类:
    1). 基于文件型
    通信进程都运行在同一主机的网络环境下，
    socket 是基于本地文件系统支持的。
    一个 socket 会关联一个特殊的文件，
    通信双方通过对这个特殊文件的读 / 写而实现通信，类似管道。

    2). 基于网络型
    非对称方式通信，发送者需要提供接收者的名称。
    通信双方运行在不同主机的网络环境下。
    通信被分配到了一对 socket,
    一个属于接收进程 (服务器端), 一个属于发送进程 (客户端)。
    一般、客户端发送连接请求时，会随机申请一个 socket, 此时主机为之分配一个端口，并使其与该套接字绑定，再不分给其他进程。

    接收进程 have_ FTP 服务器的监听端口号 21,HTTP~80

    确保了通讯双方之间逻辑链路的唯一性，便于实现数据传输的并发功能。

    1. 远程过程调用 & 远程方法调用
      远程过程调用 (remote procedure call,RPC)_ 通信协议，用于通过网络连接的系统
