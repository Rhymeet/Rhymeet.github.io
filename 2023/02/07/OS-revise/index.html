# CH1. 操作系统引论

# OS 的目标、作用、特征、功能。

# 目标：

① 方便性 ② 有效性 ③ 可扩充性 ④ 开放性

# 作用：

    1. 作为用户与计算机硬件系统之间的接口
    1. 作为计算机系统之间的管理者
    1. 实现对计算机资源的抽象

# 特征：并发、共享、虚拟、异步

虚拟：操作系统会把一个物理上的实体变为若干个逻辑上的对应物，物理实体是实际存在的，而逻辑对应物是用户感受到的.
    “空分复用技术” (让内存看起来很多) & “时分复用技术” (让CPU看起来很多)

异步：在多道进程运行的环境下，进程之中指令的执行不是一贯到底的，而是走走停停，以不可预知的速度向前推进，这就是异步性。

# 特征 --- 单道批处理系统、多道批处理系统、分时系统、实时系统。

# 单道批处理系统

系统对作业的处理是成批进行的，但内存中始终只运行一道作业

自动性(磁带上的一批作业能自动运行，无需人工干预)
顺序性(磁带上的各道作业按顺序进入内存,先调入内存的作业先运行)
单道性(内存中仅运行一道作业)

# 多道批处理系统

允许多个程序同时进入内存并运行。当一道程序因 I/O 请求而暂停时，CPU 便立即赚取运行零一道程序。达到了系统各部件之间的并行工作，提高资源利用率，增大了系统吞吐量。

多道:计算机内存中存放多道相互独立的程序
宏观上并行：同时进入系统的 多道程序都处于运行过程中，
		即 它们先后开始了各自的运行，但都未运行完毕。
微观上串行：内存中的多道程序轮流占有CPU，交替执行。

# 分时系统

比如信息查询系统
1）人机交互 2）共享主机

# (1) 多路性：

系统允许将多台终端同时连接到一台主机上,并按分时原则为每个用户提供服务。
允许多个用户共享一台主机,提高了资源利用率,降低了费用,促进了jsj广泛应用。

# (2) 独立性：

指系统提供了一种环境,即 每个用户在各自的终端上进行操作,彼此间互不干扰,给用户的感觉就是他一人独占主机。

# (3) 及时性：

用户的请求能在很短的时间内获得相应。

这一时间间隔是人们能接受的，通常 1~3 秒。

# (4) 交互性：

用户可以通过终端与 系统进行广泛的人机对话。

表现在：用户可以请求系统进行多方面的服务，如文件编辑和数据处理，访问系统中的文件系统和数据库系统，打印运行结果等。

# 实时系统

# (1) 多路性：

系统周期性的对多路现场信息进行采集,并对多个对象or执行机构进行控制

# (2) 独立性：

对信息的采集&对 对象的控制也是彼此互不干扰的

# (3) 及时性：

多媒体系统实时性 要求播出来的音频能令人满意。
实时控制系统的实时性则是以控制对象所要求的截止时间来确定的,一般为秒级~毫秒级。

# (4) 交互性：

在信息查询系统中,人与系统的交互性仅限于访问系统中的某些特定的专用服务程序。
它并不能像 分时系统那样向终端用户提供 数据处理、资源共享等服务.
多媒体系统的交互性也仅限于 由用户发送某些特定的指令,如开始、停止、快进、然后由系统立即响应。

# (5) 可靠性:

分时系统要求可靠，
实时系统要求高度可靠,因为任何差错都会带来无法预料的灾难性后果.
因此,在实时系统中,往往都采取了多级容错措施,以保障系统及数据安全。

# 概念的区别 --- 并发与共享

# 并发

-> 并行与并发
并行指 两个或多个事件在同一时刻发生；并发 ：~ 在同一时间间隔内发生

在多道程序运行环境下，并发是指一段时间内宏观上有多个程序在运行；
但在单处理机系统中，每一时刻仅能有一道程序执行，故在微观上这些程序只能分时交替执行。

并发：同一时间段内，系统可以运行多个进程。
并发 = 宏观上并行 + 微观上串行
我们 QQ,WX, 音乐。。。看来是 同时运行的 (宏观)
实际上，3 个进程是不断的切换运行的，CPU 按照某种算法切换进程 (围观)

# 共享

OS 中的资源 可供内存中的 多个并发运行的进程 共同使用。(资源复用)

宏观上既限定了时间(进程在内存中时),又限定了地点(内存)
同时使用QQ,WX，它们是并发运行的，可以共享的访问硬盘资源
    1. 互斥共享：

当进程 A 要访问资源时，必须先提出请求。
若此时该系统资源空闲，系统便可将其分配给进程 A 使用。
此后若再有其它进程要访问该资源，只要 A 未用完，其它进程必须等待。
仅当 A 访问完并且释放后，才允许另一进程对该资源访问。

摄像头资源是互斥共享的。在同一时间段，摄像头资源只能分配给一个进程。

    1. 同时共享：
      （‘同时’- 宏观）在同一时间段内，磁盘资源可以供多个进程使用。

并发与共享是多用户 OS 最基本两个特征。
互为对方存在的条件，
一方面，资源共享是以进程的并发执行为条件的，若系统不允许并发执行，则不存在资源共享问题。
另一方面，若系统不能对资源共享实施有效的管理，以协调好各进程对共享资源的访问，则必然会影响各进程间并发执行的程度，甚至根本无法并发执行。

# 系统调用的概念

计算机的各种硬件资源是有限的，为了更好的管理这些资源，用户进程是不允许直接操作的，所有对这些资源的访问都必须由操作系统控制。为此操作系统为用户态运行的进程与硬件设备之间进行交互提供了一组接口，这组接口就是所谓的系统调用。

系统调用实质上就是函数调用，只不过调用的是系统函数，处于内核态而已。 用户在调用系统调用时会向内核传递一个系统调用号，然后系统调用处理程序通过此号从系统调用表中找到相应的内核函数执行，最后返回。

是一种特殊的过程调用：-> 与一般的过程调用区别在 ->

(1) 运行在不同的系统状态
一般：
调用程序 & 被调用程序 运行在相同的状态，内核态 or 用户态
系统调用：
调用程序运行在用户态，被调用程序运行在内核态。

(2) 状态的转换
